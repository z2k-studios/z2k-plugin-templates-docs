Building URIs

In practice, most Z2K Templates URIs are generated by automation tools rather than written by hand. This page covers practical approaches for constructing URIs from different environments.

Barebones URI​

Before building a full URI, verify that the URI mechanism works at all. Copy and paste this URI directly into a web browser address bar and press Enter:

obsidian://z2k-templates?cmd=new&templateContents=Hello%20World!&fileTitle=Hello%20World&prompt=none&finalize=true

Obsidian should open (and may ask for permission to handle obsidian:// links the first time), then create a note called "Hello World" in your vault root with the text "Hello World!". No template file, no vault parameter, no configuration needed.

If this works, your URI pipeline is functioning. If it fails, check:

  • Browser permission — Your browser must be allowed to open obsidian:// links. On first use, you'll see a confirmation dialog. If you dismissed it, re-trigger or check your browser's allowed protocols.
  • Plugin enabled — Confirm Z2K Templates is active in Obsidian's Community Plugins settings.
  • Obsidian open — On some platforms, Obsidian must already be running for URI links to route correctly.
  • Vault — If you have multiple vaults and the wrong one opens, add &vault=YourVaultName to the URI.

From Apple Shortcuts​

Apple Shortcuts can build a URL by concatenating strings and using the "URL Encode" action for each parameter value. For a complete walkthrough, see How-to Pass Create Files from Templates in Apple Shortcuts.

From a Shell Script​

A reusable shell script for creating notes – useful for cron jobs, Raycast scripts, or Alfred workflows.

#!/bin/bash
# Create a meeting note with pre-filled attendees (macOS)
VAULT="My%20Vault"
TEMPLATE="Templates%2FMeeting.md"
TITLE=$(python3 -c "import urllib.parse; print(urllib.parse.quote('Standup'))")
ATTENDEES=$(python3 -c "import urllib.parse; print(urllib.parse.quote('Alice, Bob'))")

open "obsidian://z2k-templates?vault=${VAULT}&cmd=new&templatePath=${TEMPLATE}&prompt=remaining&fileTitle=${TITLE}&attendees=${ATTENDEES}&type=standup"

The open command (macOS) launches the URI. On Windows, use start; on Linux, use xdg-open.

note

For values that don't contain special characters, you can skip the encoding step and write them directly as percent-encoded strings (e.g., My%20Vault). Use the encoding helper for dynamic values or values with unpredictable characters.

From Python​

Python's urllib.parse module handles URI encoding. This example constructs a URI and opens it:

import urllib.parse
import subprocess
import platform

# Build the URI
base = "obsidian://z2k-templates"
params = {
    "vault": "My Vault",
    "cmd": "new",
    "templatePath": "Templates/Meeting.md",
    "prompt": "remaining",
    "fileTitle": "Weekly Sync",
    "attendees": "Alice, Bob, Charlie",
    "topic": "Q4 Planning"
}
uri = f"{base}?{urllib.parse.urlencode(params)}"

# Open it (cross-platform)
if platform.system() == "Darwin":
    subprocess.run(["open", uri])
elif platform.system() == "Windows":
    subprocess.run(["start", uri], shell=True)
else:
    subprocess.run(["xdg-open", uri])

The urlencode() function handles all special characters automatically.

From JavaScript​

In Node.js or browser environments, use URLSearchParams to build the query string:

// Build the URI
const base = "obsidian://z2k-templates";
const params = new URLSearchParams({
  vault: "My Vault",
  cmd: "new",
  templatePath: "Templates/Meeting.md",
  prompt: "remaining",
  fileTitle: "Weekly Sync",
  attendees: "Alice, Bob, Charlie",
  topic: "Q4 Planning"
});
const uri = `${base}?${params.toString()}`;

// In a browser, open it directly
window.location.href = uri;

// In Node.js, use the open package or child_process
// const open = require('open');
// await open(uri);

From a Browser Bookmark​

A bookmark with a obsidian://z2k-templates?... URL works as a one-click template launcher. Useful for daily notes, meeting templates, or any note you create repeatedly with the same structure.

obsidian://z2k-templates?vault=MyVault&cmd=new&templatePath=Templates%2FDaily.md

Save this as a browser bookmark. Clicking it opens Obsidian, switches to "MyVault", and creates a new note from the Daily template with full interactive prompting.

Browser Bookmarklet with Clipboard​

A bookmarklet can capture dynamic data – like text you've selected or copied – and pass it to a template. This example inserts a clipping into your daily log.

The Block Template​

Create a block template at Blocks/Web Clipping.md:

---
z2k_template_type: block
---
### {{time}}
**Source:** {{source}}

{{content}}

The Daily Log Template​

Your daily log template at Templates/Daily Log.md might look like:

---
z2k_template_type: note
z2k_dest_folder: Journal
---
# {{date}}

## Clippings

The Bookmarklet​

Create a bookmark with this JavaScript as the URL (all on one line):

javascript:(function(){
  const vault = "MyVault";
  const block = "Blocks/Web Clipping.md";
  const daily = "Journal/" + new Date().toISOString().slice(0,10) + ".md";
  const source = encodeURIComponent(window.location.href);
  const content = encodeURIComponent(window.getSelection().toString() || "No selection");
  const uri = "obsidian://z2k-templates?vault=" + vault
    + "&cmd=insertblock"
    + "&templatePath=" + encodeURIComponent(block)
    + "&existingFilePath=" + encodeURIComponent(daily)
    + "&location=file-bottom"
    + "&prompt=none"
    + "&finalize=true"
    + "&source=" + source
    + "&content=" + content;
  window.location.href = uri;
})();

How it works:

  1. Select text on any webpage
  2. Click the bookmarklet
  3. The selected text and page URL are captured
  4. A new clipping block is inserted at the bottom of today's daily log

The bookmarklet constructs the daily log filename using today's date (e.g., Journal/2025-03-15.md). Adjust the path pattern to match your vault structure.

note

Bookmarklets have a URL length limit (varies by browser, typically 2000+ characters). For very long selections, the URI may be truncated. Consider using the fieldData directive with Base64 encoding for large content, or limit the selection length in the bookmarklet.

INTERNAL NOTES
  • The shell script example uses python3 -c "import urllib.parse; ..." for encoding. This is portable but requires Python 3. Alternative: jq -rn --arg v "value" '$v|@uri' for systems with jq installed.
  • On iOS, URIs are triggered through the Shortcuts app or by tapping a link. The open command is macOS-specific.
  • The bookmarklet example uses window.getSelection() which captures selected text on the page, not clipboard contents. True clipboard access requires the Clipboard API and user permission, which is unreliable in bookmarklets.
  • How-To Guide: The Browser Bookmarklet with Clipboard section has a companion how-to guide at How to Clip Web Content into Your Daily Log with a Browser Bookmarklet. When the how-to guides go public, trim this section to a brief description and link there.
