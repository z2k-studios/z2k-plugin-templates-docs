field-info Cheat Sheet

field-info Cheat Sheet​

The {{field-info}} built-In helper function is a powerful tool for controlling how a {{fieldName}} is prompted for and handled by the plugin. There is a lot behind this workhorse of a helper. To help you get a one-page overview, here is a quick cheat sheet on its core functions.

Syntax​

The Syntax of {{field-info}} supports both positional and named parameters:

  • Positional Parameters:
    • {{field-info fieldName prompt default type}}
  • Named Parameters:
    • {{field-info fieldName=fieldName prompt=prompt default=default type=type opts=opts miss=miss directives=*directives}}

Parameters​

Overview

There are a number of parameters that can be used with {{field-info}}:

Positional + Named Parameters:

  • fieldName :: the name of the field you are providing data for (required)
  • prompt :: The prompt message to display; can use {{fields}} inside it
  • default :: The default field result to prefill in the prompt dialog box. Only accepted if "touched".
  • type :: Type of field ("text", "number", "date", "datetime", "boolean", "singleSelect", "multiSelect", "titleText";)

Named Only Parameters:

  • opts :: A comma separated string list of available options for singleSelect and multiSelect types
  • miss :: The field result that will be used if the use never attempts to answer the prompt
  • directives :: A comma separated list of directives for advanced usage
  • value :: An advanced use case where you pre-fill a value into a field, skipping prompting

To understand the difference between positional vs. named parameter usage, please see field-info Syntax.

Directives​

Overview​

The directives parameter is a list of strings, with each string being one of the following directives:

  • Miss Handling Special Cases:
    • 'finalize-default' :: Instructs the plugin to use the value of the default string as the resultant value of the field if the user has not provided an answer
    • 'finalize-preserve' :: Instructs the plugin to preserve the {{field}} entry in the final generated file if the user does not specify a value
    • 'finalize-clear' :: Directs the plugin to clear out the value of this field if no value is provided
  • Required:
    • 'required' :: Tells the Prompting Interface that a field requires a value in order to finalize the card
    • 'not-required' :: Tells the Prompting Interface that a field does not require a value in order to finalize the card
  • Prompting:
    • 'no-prompt' :: Directs the Prompting Interface to skip prompting the user for a field
    • 'yes-prompt' :: Directs the Prompting Interface to force the prompting of a user for a field
